# Task 3.4 -- Local Workspace Manager

## Objective

Manage per-session local workspaces: create a directory, clone the project repo from the Docker-mounted `/repo` path, and clean up after session completion.

## Context

NanoClaw runs in Docker. The container mounts the project repo at `/repo`. Each headless agent session gets its own workspace directory under `/workspaces/<sessionId>` via `git clone`. Docker provides the isolation boundary — no external sandbox service needed.

Originally this task used Daytona SDK for remote sandboxes. Rewritten in the lean integration overhaul to use local directories.

## Requirements

- [x] `SandboxDeps = { baseDir: string }` — base directory for workspaces
- [x] `SandboxHandle = { sessionId: string, workDir: string }`
- [x] `createSandbox(deps, config)` — mkdir + git clone from local repo mount
- [x] `destroySandbox(deps, sessionId)` — rm -rf + map cleanup
