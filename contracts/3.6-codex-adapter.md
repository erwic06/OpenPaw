# Task 3.6 -- Codex Adapter

## Objective

Implement a CodexAdapter that spawns OpenAI Codex sessions via `@openai/codex-sdk`, mirroring the LLMAdapter pattern. Replaces the Daytona MCP tools and OpenAI chat completion adapter.

## Context

The lean integration overhaul replaced Daytona (remote sandbox) and the OpenAI raw API adapter with two first-class agent harnesses: Claude Code (LLMAdapter) and Codex (CodexAdapter). Both use native file/shell/git tools — no MCP wiring needed.

Tier-based routing: heavy/standard → Claude Code primary, Codex fallback. Light → Codex primary, Claude fallback. Cross-provider fallback on retryable errors.

## Requirements

- [x] `CodexAdapter` implements `AgentAdapter` (trigger/status/output/cancel)
- [x] Spawns Codex sessions via `codex.startThread()` + `thread.runStreamed()`
- [x] Iterates JSONL events for real-time usage tracking and activity timestamps
- [x] Maps `turn.completed` events to token usage and cost logging
- [x] `waitForCompletion()` and `getLastActivityMs()` for session monitoring
- [x] DI via `codexFactory` for testing (avoids real Codex subprocess in tests)
- [x] Model mapping: light → gpt-5.4-mini, standard/heavy → gpt-5.4
- [x] `cwd` option sets workspace directory
- [x] `sandboxMode: "danger-full-access"`, `approvalPolicy: "never"` for headless operation
