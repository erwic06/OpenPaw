# Task 8.4 -- ServiceAdapter

## Objective

Implement the `ServiceAdapter` class that wraps HTTP calls to external services, implementing the `AgentAdapter` interface alongside the existing LLMAdapter, CodexAdapter, and GeminiAdapter.

## Context

The design doc specifies two adapter types: LLM-native (API sessions against LLM providers) and external services (standalone services with HTTP APIs). LLMAdapter, CodexAdapter, and GeminiAdapter handle LLM-native agents. The ServiceAdapter handles external service agents — it triggers work via HTTP, polls for status, and retrieves output.

Depends on 8.1 (types, specifically AdapterConfig). Independent of scheduling (8.5-8.6) and routing (8.7).

## Requirements

- [ ] `ServiceAdapter` class implements `AgentAdapter` interface from `src/agents/types.ts`
- [ ] `ServiceAdapterDeps` interface: `{ db: Database; config: ServiceAdapterConfig; fetchFn?: typeof fetch; secrets?: Map<string, string> }`
- [ ] `ServiceAdapterConfig` from AdapterConfig service variant: baseUrl, auth, healthCheck, triggerEndpoint, statusEndpoint, outputEndpoint
- [ ] `trigger(input: AgentInput): Promise<string>`:
  - POST to `config.triggerEndpoint` with AgentInput payload
  - Insert session row in DB
  - Return session ID (from response or generated with `service-` prefix)
- [ ] `status(sessionId: string): Promise<AgentStatus>`:
  - GET `config.statusEndpoint` with `{session_id}` template substitution
  - Map response to AgentStatus
- [ ] `output(sessionId: string): Promise<AgentOutput>`:
  - GET `config.outputEndpoint` with `{session_id}` template substitution
  - Map response to AgentOutput
- [ ] `cancel(sessionId: string): Promise<void>`:
  - POST/DELETE to cancel endpoint (configurable)
- [ ] `healthCheck(): Promise<boolean>`:
  - GET `config.healthCheck` endpoint
  - Return true if 200 response, false otherwise
- [ ] Auth handling: if `config.auth` starts with `${` and ends with `}`, resolve from secrets Map; otherwise use as-is
  - Bearer token: `Authorization: Bearer <token>`
- [ ] HTTP timeout: 30s default for status/output, 60s for trigger
- [ ] Error mapping: 5xx → retryable, 4xx → non-retryable, timeout → retryable
- [ ] DI via `fetchFn` for testing without real HTTP calls

## Interface Contract

**Inputs:** AgentInput for trigger, session IDs for status/output/cancel.

**Outputs:** AgentStatus, AgentOutput following existing adapter patterns.

**Side Effects:** HTTP requests to external services. Session rows in SQLite.

## Technical Approach

```typescript
// src/agents/service-adapter.ts
export class ServiceAdapter implements AgentAdapter {
  constructor(private deps: ServiceAdapterDeps) {}
  
  async trigger(input: AgentInput): Promise<string> {
    const sessionId = `service-${crypto.randomUUID()}`;
    const url = `${this.deps.config.baseUrl}${this.deps.config.triggerEndpoint}`;
    const res = await this.fetch(url, {
      method: "POST",
      body: JSON.stringify({ taskId: input.taskId, input: input.input }),
    });
    if (!res.ok) throw new Error(`Service trigger failed: ${res.status}`);
    // Insert session row
    return sessionId;
  }
  
  private async fetch(url: string, init?: RequestInit): Promise<Response> {
    const fetchFn = this.deps.fetchFn ?? fetch;
    const auth = this.resolveAuth();
    return fetchFn(url, {
      ...init,
      headers: { "Authorization": `Bearer ${auth}`, "Content-Type": "application/json", ...init?.headers },
      signal: AbortSignal.timeout(init?.method === "POST" ? 60000 : 30000),
    });
  }
}
```
