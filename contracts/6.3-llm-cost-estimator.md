# Task 6.3 -- LLM Cost Estimator

## Objective

Create an async LLM-based cost estimator that uses a Haiku-tier model to estimate research cost more accurately than the static DEPTH_CONFIGS table, with graceful fallback to the existing static estimator on failure.

## Context

The current cost estimation in `src/research/estimator.ts` uses `DEPTH_CONFIGS` lookup tables (task 4.7). The design doc (lines 1192-1207) calls for a lightweight Haiku-tier LLM call that considers the actual prompt content when estimating costs. The `ResearchRunnerDeps` has a sync `estimateCostFn` DI slot; this new async estimator lives in the intake layer (task 6.5 calls it before invoking ResearchRunner). ResearchRunner's sync fallback is unchanged.

No dependencies. Blocks 6.5 (research intake).

## Requirements

- [ ] `LLMCostEstimatorDeps` interface with DI:
  - `anthropicApiKey: string`
  - `queryFn?: (systemPrompt: string, userPrompt: string) => Promise<string>` -- override for testing
- [ ] `estimateCostWithLLM(deps, depth, prompt, model?): Promise<CostEstimate>` function
- [ ] Uses `claude-haiku-4-5` via the Anthropic Messages API (direct fetch, not Agent SDK -- this is a single-turn API call)
- [ ] Prompt to Haiku: system prompt explains DEPTH_CONFIGS baseline, asks for JSON output with `estimatedTokens` and `estimatedCostRange: [low, high]`
- [ ] Parses JSON from response, validates fields, returns `CostEstimate`
- [ ] Falls back to `estimateResearchCost()` from `src/research/estimator.ts` on any failure (network error, parse error, invalid response)
- [ ] Re-exports `CostEstimate` type from `src/research/estimator.ts`
- [ ] Exported from `src/intake/index.ts` barrel

## Interface Contract

**Inputs:** `depth` (1-10), `prompt` (research topic string), optional `model` override.

**Outputs:** `Promise<CostEstimate>` -- same type as the existing sync estimator.

**Side Effects:** One API call to Anthropic Messages API (Haiku tier).

## Technical Approach

```typescript
// src/intake/estimator.ts
import { estimateResearchCost, type CostEstimate } from "../research/estimator.ts";
import { DEPTH_CONFIGS } from "../research/types.ts";

export type { CostEstimate };

export interface LLMCostEstimatorDeps {
  anthropicApiKey: string;
  queryFn?: (systemPrompt: string, userPrompt: string) => Promise<string>;
}

export async function estimateCostWithLLM(
  deps: LLMCostEstimatorDeps,
  depth: number,
  prompt: string,
  model = "claude-haiku-4-5",
): Promise<CostEstimate> {
  try {
    const config = DEPTH_CONFIGS[depth];
    const systemPrompt = `You estimate research costs. Given a depth level and prompt, return JSON...`;
    const userPrompt = `Depth: ${depth}\nBaseline tokens: ${config.outputTokenBudget}\nPrompt: ${prompt}`;

    const queryFn = deps.queryFn ?? defaultQuery(deps.anthropicApiKey, model);
    const response = await queryFn(systemPrompt, userPrompt);
    const parsed = JSON.parse(response);

    // Validate and return
    return {
      depth,
      estimatedTokens: parsed.estimatedTokens,
      estimatedCostRange: [parsed.estimatedCostRange[0], parsed.estimatedCostRange[1]],
    };
  } catch {
    // Fallback to static estimation
    return estimateResearchCost(depth, prompt);
  }
}
```

The `defaultQuery` function uses `fetch("https://api.anthropic.com/v1/messages", ...)` with the Anthropic Messages API format. This avoids importing the full Agent SDK for a single-turn call.

**Key decision:** This does NOT modify `ResearchRunnerDeps.estimateCostFn` (which is sync). The intake layer (6.5) calls `estimateCostWithLLM` directly, then passes the result to the spend gate. ResearchRunner still has its own sync fallback for when tasks are dispatched via the plan watcher (non-intake path).
