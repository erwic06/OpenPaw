# Task 6.2 -- Planner Agent System Prompt

## Objective

Create a system prompt for the Planner agent that can operate in two modes: project planning (scaffolding a new project repo) and research planning (breaking a deep research prompt into sub-tasks).

## Context

The Planner agent is referenced in the design doc Section 1 filesystem matrix. It is spawned by the `/project` intake flow (task 6.6) and the `/dr` deep research flow (task 6.5). Both flows use the Agent SDK with Claude Sonnet and this system prompt.

No dependencies. Blocks 6.6 (project intake).

## Requirements

- [ ] File: `agents/planner/system_prompt.md`
- [ ] Two clearly delineated modes:
  - **Project planning mode:** Read the user's project description. Create a phased `implementation_plan.md` with tasks, dependencies, and contracts per task. Scaffold `CLAUDE.md`, `project_spec.md`, and contract files. Follow the format of the existing OpenPaw `implementation_plan.md` and `contracts/_template.md`.
  - **Research planning mode:** Break a research prompt into 3-8 sub-tasks at depth 8-10. Each sub-task has a clear scope, target sources, and estimated depth. Output is a structured research plan.
- [ ] Filesystem constraints:
  - Read: `project_spec.md`, `CLAUDE.md` (templates)
  - Read-write: `implementation_plan.md`, `contracts/*`
  - No access: `src/*`, `research/*`, secrets, `.env`
- [ ] Output format: structured markdown with task IDs, dependencies, descriptions
- [ ] Anti-sycophancy: challenge vague prompts, push back on overambitious scope
- [ ] No code execution, no BrowserUse, no external API calls

## Interface Contract

**Inputs:** User prompt (project description or research topic), mode indicator (project or research).

**Outputs:** `agents/planner/system_prompt.md` file.

**Side Effects:** None (content-only task).

## Technical Approach

Structure the prompt in sections:
1. **Identity and role:** You are a project planner for the OpenPaw agent fleet.
2. **Mode detection:** If the prompt starts with `[PROJECT]`, use project planning mode. If it starts with `[RESEARCH]`, use research planning mode. The intake layer prefixes the mode tag.
3. **Project planning:** Read the description, derive project name, create phased plan with tasks grouped by milestone. Each task gets: ID, title, type, dependencies, acceptance criteria. Write contracts following the template format.
4. **Research planning:** Break the prompt into sequential sub-tasks. Each sub-task: title, scope (what to investigate), depth (8-10), target source count, dependency on prior sub-tasks (for building on earlier findings).
5. **Quality gates:** Before finalizing, check: Are dependencies acyclic? Are task scopes non-overlapping? Is the plan achievable? Push back if the prompt is too vague.
