# Task 4.6 -- Research Fact-Check Reviewer

## Objective

Create an adversarial fact-checking Reviewer for research briefs. Distinct from the code reviewer (Task 3.10). Reuses the existing `ReviewResult`/`ReviewFinding` types and `parseReviewResult` function.

## Context

Cross-model review is a key sycophancy mitigation: Gemini produces the research, Claude (Sonnet) reviews it. The fact-checker focuses on: factual accuracy, source quality, unsupported claims, citation completeness. The review infrastructure from Task 3.10 (types, parsing, session execution pattern) is reused.

Depends on Task 4.5 (research types for prompt building).

## Requirements

- [ ] `agents/researcher-reviewer/system_prompt.md` -- adversarial fact-checking prompt:
  - Review focus: factual accuracy, source quality, citation completeness, logical coherence
  - Severity mapping: critical=fabricated claim with citation, major=unsupported assertion, minor=weak source, nit=formatting
  - Decision rules: same pattern as code reviewer (critical -> REQUEST_CHANGES, 2+ major -> REQUEST_CHANGES)
  - Output format: same JSON structure as code reviewer (`ReviewResult`)
  - Constraints: review the brief only, do not rewrite, do not expand scope
- [ ] `runResearchReview(deps, systemPromptPath, cwd, taskId, brief)` function
  - Analogous to `runCodeReview` but takes research brief text instead of git diff
  - Uses `ReviewExecutor` DI for testing
  - Logs session and cost to SQLite
  - Returns `ReviewResult | null` (null on session failure -- soft pass)
- [ ] `buildResearchReviewPrompt(taskId, brief)` -- formats the review prompt
- [ ] Reuses `parseReviewResult` from `src/review/index.ts` (same JSON output format)
- [ ] Unit tests with mocked executor

## Interface Contract

**Inputs:** Research brief text, task metadata.

**Outputs:** `ReviewResult` with fact-check findings, or null on failure.

**Side Effects:** SQLite session/cost records.

## Technical Approach

The implementation closely mirrors `runCodeReview` in `src/review/index.ts`. Key differences:
- Prompt contains a research brief instead of a git diff
- System prompt focuses on factual accuracy instead of code quality
- The `file` field in `ReviewFinding` holds the source URL (not a file path)
- The `line` field is unused (set to 0)
