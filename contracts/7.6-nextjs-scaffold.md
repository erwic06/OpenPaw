# Task 7.6 -- Next.js Project Scaffold and Layout Shell

## Objective

Create the Next.js App Router project in `web/` with Tailwind CSS dark theme, persistent layout shell (vertical tab bar, header with command input and daily spend badge), and typed API/WebSocket client libraries.

## Context

The OpenPaw web dashboard is a separate Next.js application deployed to Vercel. It consumes the NanoClaw REST API and WebSocket endpoints. This task creates the project scaffold and the persistent layout — the individual tab pages are built in tasks 7.7-7.9.

No backend dependencies — can be built in parallel with tasks 7.1-7.4. The frontend uses Tailwind CSS for styling with a dark theme and amber/gold accent color, following the design doc's UI specification.

## Requirements

- [ ] Next.js App Router project in `web/` directory with `package.json`, `tsconfig.json`, `next.config.ts`
- [ ] Tailwind CSS v4 configured with dark theme (dark background, amber/gold accent)
- [ ] Root layout (`web/src/app/layout.tsx`) with persistent sidebar and header
- [ ] Vertical tab bar component (`web/src/components/sidebar.tsx`) with 5 tabs:
  - Overview (icon + label)
  - Research (icon + label)
  - Projects (icon + label)
  - Content Review (icon + label)
  - Automations (icon + label, placeholder for Phase 8)
- [ ] Header component (`web/src/components/header.tsx`) with:
  - Command input bar ("Ask anything or give a task")
  - Daily spend badge (fetches from API)
- [ ] Active tab highlighting in sidebar based on current route
- [ ] API client (`web/src/lib/api.ts`): typed fetch wrapper with base URL configuration
- [ ] WebSocket client (`web/src/lib/ws.ts`): reconnecting WebSocket with typed event dispatch
- [ ] Environment variable: `NEXT_PUBLIC_API_URL` (defaults to `http://localhost:9999`)
- [ ] `web/.gitignore` for Next.js (node_modules, .next, etc.)
- [ ] `web/src/app/page.tsx` redirects to `/overview`
- [ ] Automations tab shows "Coming soon" placeholder

## Interface Contract

**Inputs:** `NEXT_PUBLIC_API_URL` environment variable.

**Outputs:** Runnable Next.js application with layout shell. API and WebSocket client libraries.

**Side Effects:** None (static scaffold).

## Technical Approach

// web/src/lib/api.ts
const BASE_URL = process.env.NEXT_PUBLIC_API_URL || "http://localhost:9999";

export async function apiFetch<T>(path: string, options?: RequestInit): Promise<T> {
  const res = await fetch(`${BASE_URL}${path}`, {
    ...options,
    headers: {
      "Content-Type": "application/json",
      ...options?.headers,
    },
  });
  if (!res.ok) {
    const err = await res.json().catch(() => ({ error: res.statusText }));
    throw new ApiError(err.error || res.statusText, res.status);
  }
  return res.json();
}

## What NOT to Do

- Do not implement actual page content -- that is tasks 7.7-7.9
- Do not install 21st.dev components yet unless they're straightforward to set up -- focus on Tailwind
- Do not add authentication handling in the frontend -- Cloudflare Access handles auth transparently
- Do not add state management libraries (Redux, Zustand) -- React state + context is sufficient for single-user app

## Recovery Guidance

- If Tailwind v4 has breaking changes, fall back to v3 with postcss config
- The API client should handle the case where the backend is not running (show error state)
- If `npm run build` fails on type errors, check that `tsconfig.json` has `"strict": true` and `"jsx": "preserve"`
