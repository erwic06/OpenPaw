# Task 8.9 -- Template Agent Definitions and E2E Verification

## Objective

Create 3 template agent definitions exercising cron, event, and manual trigger modes, and verify the full Phase 8 pipeline end-to-end.

## Context

The fleet system is fully implemented (8.1-8.8). This task creates real agent definitions that serve as templates and test the full roundtrip: markdown → parser → loader → DB → scheduler → output routing. It also performs end-to-end verification of the Docker build and integration.

Depends on 8.8 (fleet orchestrator).

## Technical Approach

Each agent.md follows the format specified in the design doc. The E2E test:

```typescript
// tests/fleet-e2e.test.ts
import { parseAgentDefinition } from "../src/fleet/parser.ts";
import { loadAllAgentDefinitions, syncDefinitionsToDb } from "../src/fleet/loader.ts";
import { matchesCron } from "../src/fleet/cron.ts";
import { initDatabase } from "../src/db/index.ts";

test("TBPN digest parses correctly", () => {
  const content = readFileSync("agents/tbpn-digest/agent.md", "utf-8");
  const def = parseAgentDefinition(content, "agents/tbpn-digest/agent.md");
  expect(def.id).toBe("tbpn-digest");
  expect(def.schedule.type).toBe("cron");
  expect(def.schedule.expression).toBe("0 20 * * *");
  expect(def.outputDestinations).toHaveLength(2);
});

test("TBPN triggers at 8 PM", () => {
  expect(matchesCron("0 20 * * *", new Date("2026-04-02T20:00:00"))).toBe(true);
  expect(matchesCron("0 20 * * *", new Date("2026-04-02T19:00:00"))).toBe(false);
});
```
