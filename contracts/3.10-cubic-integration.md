# Task 3.10 -- Cubic Integration

## Objective

Integrate with Cubic (GitHub App for automated PR review): after a Coder session opens a PR, poll the GitHub API for the Cubic review to appear, then capture and return the review summary.

## Context

Cubic is a GitHub App that automatically reviews PRs when opened. NanoClaw does not trigger Cubic -- it triggers automatically. NanoClaw's job is to wait for the review to appear and read the results via the GitHub REST API. The review summary is used by the deploy gate (Task 3.11) and logged for audit.

Depends on Task 3.6 (Daytona MCP tools -- the Coder creates PRs via git tools).

## Requirements

- [ ] `waitForCubicReview(owner, repo, prNumber)` -- poll GitHub API until Cubic's review appears (or timeout after 10 minutes)
- [ ] `getCubicReviewSummary(owner, repo, prNumber)` -- extract review body, findings, and pass/fail from Cubic's review
- [ ] Uses native `fetch` (Bun built-in) -- no GitHub SDK
- [ ] GitHub token loaded from Docker secrets
- [ ] Polling interval: 30 seconds
- [ ] DI pattern: `CubicDeps = { githubToken: string, fetchFn?: typeof fetch }`
- [ ] Unit tests with mocked fetch responses

## Interface Contract

**Inputs:** GitHub repo owner, repo name, PR number. GitHub token from secrets.

**Outputs:**
- `waitForCubicReview(...)` -- returns Cubic review data or null on timeout
- `getCubicReviewSummary(...)` -- returns structured summary (findings, pass/fail)

**Side Effects:** Makes GitHub API requests.

## Technical Approach

Poll `GET /repos/{owner}/{repo}/pulls/{pr}/reviews` via GitHub REST API. Filter for reviews where the author is the Cubic bot (identify by bot username or app ID). Extract the review body and state (APPROVED, CHANGES_REQUESTED, COMMENTED).

```typescript
async function waitForCubicReview(owner: string, repo: string, prNumber: number, deps: CubicDeps): Promise<CubicReview | null> {
  const deadline = Date.now() + 10 * 60 * 1000;
  while (Date.now() < deadline) {
    const reviews = await fetchReviews(owner, repo, prNumber, deps);
    const cubic = reviews.find(r => r.user.login === CUBIC_BOT_USERNAME);
    if (cubic) return parseCubicReview(cubic);
    await Bun.sleep(30_000);
  }
  return null;
}
```

## What NOT to Do

- Do not implement webhook-based review detection -- polling is simpler and sufficient
- Do not add a GitHub SDK dependency -- native fetch is enough for two endpoints
- Do not parse Cubic's review format in detail beyond what's needed for the deploy gate context

## Recovery Guidance

- If Cubic bot username changes: update the `CUBIC_BOT_USERNAME` constant
- If GitHub API rate limits: add rate limit header checking and backoff
- If Cubic review format changes: update the parser; log raw review body for debugging
