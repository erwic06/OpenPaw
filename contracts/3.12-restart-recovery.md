# Task 3.12 -- Restart Recovery

## Objective

Implement orchestrator restart recovery: on NanoClaw startup, detect orphaned sessions (started but never completed), mark them as FAILED, reset corresponding tasks in the implementation plan to "ready", and notify the human via Telegram.

## Context

The design doc (Section 4, F5) specifies: on startup, query SQLite for sessions with `started_at IS NOT NULL AND ended_at IS NULL`. These are sessions that were interrupted by a NanoClaw restart (Mac Mini reboot, Docker restart, crash). The plan must be updated so these tasks are re-dispatched.

Depends on Task 3.9 (session runner -- needs plan writer utility).

## Requirements

- [ ] `recoverOrphanedSessions()` runs at NanoClaw startup, before the plan watcher starts
- [ ] Query SQLite: `SELECT * FROM sessions WHERE ended_at IS NULL`
- [ ] For each orphaned session:
  - Update session: `ended_at = now, terminal_state = 'FAILED', error = 'orchestrator restart, session orphaned'`
  - Find the corresponding task in `implementation_plan.md`
  - Update task status from `in-progress` to `ready` with note: "prior session interrupted by restart"
- [ ] If any orphaned sessions found: send Telegram alert listing affected task IDs
- [ ] Called from `src/index.ts` at startup, after DB init but before plan watcher starts
- [ ] Unit tests

## Interface Contract

**Inputs:** Database handle, plan file path, messaging function.

**Outputs:** Number of recovered sessions.

**Side Effects:** Updates SQLite session records. Modifies `implementation_plan.md`. Sends Telegram alert.

## Technical Approach

```typescript
async function recoverOrphanedSessions(deps: RecoveryDeps): Promise<number> {
  const orphaned = getOrphanedSessions(deps.db);
  if (orphaned.length === 0) return 0;

  for (const session of orphaned) {
    updateSession(deps.db, session.id, {
      ended_at: new Date().toISOString(),
      terminal_state: 'FAILED',
      error: 'orchestrator restart, session orphaned'
    });
    if (session.task_id) {
      updateTaskStatus(deps.planPath, session.task_id, 'ready',
        'prior session interrupted by restart');
    }
  }

  await deps.sendAlert(...);
  return orphaned.length;
}
```

Needs a new DB query: `getOrphanedSessions(db)` -- `SELECT * FROM sessions WHERE ended_at IS NULL`.

## What NOT to Do

- Do not attempt to resume interrupted sessions -- just clean up and re-dispatch
- Do not delete orphaned session records -- update them for audit trail
- Do not clean up Daytona sandboxes here -- those are independent resources

## Recovery Guidance

- If plan writer fails during recovery: the session is still marked in SQLite; next restart will not double-process it (ended_at is set)
- If many sessions are orphaned: they are processed sequentially; this is fine since it runs once at startup
