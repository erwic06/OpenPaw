# Task 7.8 -- Research and Projects Pages with Session Panels

## Objective

Build the Research tab (active/completed research list) and Projects tab (project list with workspace view featuring tmux-like tiled session panels with real-time streaming).

## Context

The Research tab shows research tasks with their status, depth, model, and cost. The Projects tab is the most complex UI: a project list view that opens into a workspace with a secondary task sidebar and tiled, resizable session panels. Each session panel connects to a WebSocket for real-time output streaming.

Depends on 7.6 (scaffold) and 7.2 (REST endpoints).

## Requirements

### Research Tab
- [ ] Research page at `web/src/app/research/page.tsx`
- [ ] Active research section: task name, depth, model, cost, duration, status indicator
- [ ] Completed research section: same fields plus summary preview
- [ ] Clicking active research opens a session panel with live streaming

### Projects Tab
- [ ] Projects list at `web/src/app/projects/page.tsx`
- [ ] Each project card: name, description, completion stats, progress bar, [Open] button
- [ ] Project workspace at `web/src/app/projects/[id]/page.tsx`
- [ ] Secondary sidebar: task list with status indicators (checkmark/dot/circle)
- [ ] Main area: tiled session panels

### Session Panel Component
- [ ] `session-panel.tsx`: reusable across Research and Projects views
- [ ] Panel header: task name, model info (e.g., "[Coder, Sonnet 4.6]")
- [ ] Live session output area (scrolling text from WebSocket stream)
- [ ] Status indicator: cost, duration, progress
- [ ] HITL gate display when gate is pending (approve/edit/deny buttons)
- [ ] Close button to dismiss panel

### Panel Layout
- [ ] `panel-layout.tsx`: container for tiled session panels
- [ ] Panels are resizable (drag border between panels)
- [ ] Panels are closable (X button)
- [ ] New panels added when clicking a task in the sidebar

## Interface Contract

**Inputs:** REST API responses (sessions, projects, tasks). WebSocket session stream events.

**Outputs:** React components for Research tab, Projects tab, session panels.

**Side Effects:** WebSocket connections opened per active session panel. Gate decisions via POST.

## Technical Approach

The panel layout uses CSS Grid with draggable column/row dividers. Each session panel manages its own WebSocket connection to `/api/ws/sessions/:id/stream`. Session output is displayed as a scrolling log with auto-scroll to bottom.

For the task sidebar, use the plan parser's task data from `GET /api/projects/:id/tasks`.

## What NOT to Do

- Do not implement complex panel drag-and-drop rearrangement -- resize and close are sufficient
- Do not buffer unlimited session output -- keep last 500 lines per panel
- Do not fetch agent data from Phase 8 endpoints
