# Task 4.5 -- Research Types and Contract Template

## Objective

Define the research data types (brief, section, source, depth config) and create a research contract template for structured research output.

## Context

Research tasks produce structured briefs with citations. The depth scale (1-10) maps to token budgets and source requirements. These types are consumed by the Researcher agent (4.4), Research Reviewer (4.6), and Research Runner (4.7).

No dependencies. Foundation types for the research pipeline.

## Requirements

- [ ] `ResearchBrief` type: `{ taskId, title, depth, sections[], sources[], generatedAt, model, costUsd }`
- [ ] `ResearchSection` type: `{ heading, content, citations[] }`
- [ ] `ResearchSource` type: `{ id, url, title, accessDate, reliability }`
- [ ] `SourceReliability` type: `"official" | "academic" | "reputable" | "community" | "unknown"`
- [ ] `DepthConfig` type: `{ outputTokenBudget, minSources, estimatedCostRange: [number, number] }`
- [ ] `DEPTH_CONFIGS` constant: map from depth 1-10 to DepthConfig values
  - Depth 1-2: ~2K output tokens, 1-2 sources, $0.05-$0.15
  - Depth 3-4: ~5K output tokens, 2-3 sources, $0.15-$0.50
  - Depth 5-6: ~12K output tokens, 3-5 sources, $0.50-$1.50
  - Depth 7-8: ~25K output tokens, 5-8 sources, $1.50-$4.00
  - Depth 9-10: ~50K+ output tokens, 8+ sources, $4.00-$12.00
- [ ] `parseResearchBrief(text)` -- extract structured brief from markdown output
- [ ] Research contract template at `contracts/_research_template.md`
- [ ] Barrel exports from `src/research/index.ts`
- [ ] Unit tests for DEPTH_CONFIGS validation and brief parsing

## Interface Contract

**Inputs:** None (type definitions and constants).

**Outputs:** Exported types, DEPTH_CONFIGS, parseResearchBrief.

**Side Effects:** None.

## Technical Approach

Types in `src/research/types.ts`. The `DEPTH_CONFIGS` constant maps each depth level to its configuration. `parseResearchBrief` extracts structured data from the markdown output of a Researcher session (headings become sections, footnotes become sources).

The research contract template follows the same structure as `contracts/_template.md` but adds research-specific fields: prompt, depth, output requirements.
