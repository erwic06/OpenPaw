# Task 6.5 -- Research Intake Flow

## Objective

Implement the `/research` and `/dr` (deep research) intake flows that handle cost estimation, spend gate approval, and dispatch to the existing `ResearchRunner`, with `/dr` adding an intermediate Planner session and plan gate.

## Context

The design doc (lines 1123-1139, 1638-1649) defines two research intake paths:
- `/research <depth> <prompt>`: single session, cost gate only
- `/dr <prompt>`: deep research at depth 8-10, Planner decomposes into sub-tasks, plan gate, multi-session dispatch

Both flows reuse the existing `ResearchRunner.runResearch()` from `src/research/runner.ts` (task 4.7). The LLM cost estimator from task 6.3 provides async cost estimation before the spend gate. These are ad-hoc commands that bypass the plan file -- they create synthetic `Task` objects.

Depends on 6.1 (parser types), 6.3 (LLM cost estimator), 6.4 (dispatch handler). Blocks 6.9.

- [ ] `/research` flow:
  1. Estimate cost via `estimateCostWithLLM` (6.3), fallback to DEPTH_CONFIGS
  2. Send cost estimate echo to Telegram
  3. Request spend gate (`gateType: "spend"`)
  4. On approval: create synthetic Task, call `researchRunner.runResearch(task, depth, prompt)`
  5. On denial: send denial message
- [ ] `/dr` flow:
  1. Set depth to 9 (fixed for deep research)
  2. Estimate cost, send echo, request spend gate
  3. On spend approval: spawn Planner session (Claude Sonnet via Agent SDK) with `[RESEARCH]` prefix
  4. Send Planner output summary to Telegram
  5. Request plan gate (`gateType: "plan"`)
  6. On plan approval: parse sub-tasks from Planner output, dispatch each sequentially via `researchRunner.runResearch()`
  7. On denial at any gate: send denial message
- [ ] Synthetic Task IDs: `research-<timestamp>` or `dr-<timestamp>`
- [ ] All exports from `src/intake/index.ts` barrel

## Interface Contract

**Inputs:** Parsed command args from dispatch (depth + prompt for /research, prompt for /dr).

**Outputs:** Dispatches research sessions. Sends Telegram messages for cost echo, gate results, completion.

**Side Effects:** Creates sessions in DB (via ResearchRunner), sends Telegram messages, triggers HITL gates.

## Technical Approach

```typescript
// src/intake/research.ts
import type { Task } from "../plan/types.ts";
import type { ResearchRunner } from "../research/runner.ts";
import type { GateRequest, GateResult } from "../gates/types.ts";
import { estimateCostWithLLM, type LLMCostEstimatorDeps, type CostEstimate } from "./estimator.ts";
import { formatCostEstimate } from "../research/estimator.ts";
import { requestApproval as realRequestApproval } from "../gates/index.ts";

export function createSyntheticTask(command: string, prompt: string): Task {
  const id = `${command}-${Date.now()}`;
  const title = prompt.length > 80 ? prompt.slice(0, 80) + "..." : prompt;
  return {
    id,
    title,
    status: "in-progress",
    type: "research",
    contract: "",
    dependencies: [],
    assigned: "intake",
    artifacts: [],
    acceptance: "",
    notes: [],
  };
}

export async function handleResearch(
  deps: ResearchIntakeDeps,
  depth: number,
  prompt: string,
): Promise<void> {
  // 1. Estimate cost (async LLM, fallback to static)
  const estimate = await (deps.estimateCostFn ?? estimateCostWithLLM)(
    { anthropicApiKey: deps.anthropicApiKey },
    depth,
    prompt,
  );

  // 2. Echo cost estimate
  await deps.sendMessage(deps.chatId, formatCostEstimate(estimate, prompt));

  // 3. Spend gate
  const requestFn = deps.requestApprovalFn ?? realRequestApproval;
  const result = await requestFn({
    gateType: "spend",
    taskId: null,
    sessionId: null,
    contextSummary: formatCostEstimate(estimate, prompt),
  });

  if (result.decision !== "approved") {
    await deps.sendMessage(deps.chatId, "Research cancelled.");
    return;
  }

  // 4. Dispatch
  const task = createSyntheticTask("research", prompt);
  await deps.researchRunner.runResearch(task, depth, prompt);
}
```

For `/dr`, the Planner session follows the same Agent SDK pattern as `ResearchRunner.runClaudeFallbackSession()` -- import `query` from `@anthropic-ai/claude-agent-sdk`, iterate the async generator, collect result text. The Planner's `[RESEARCH]` prefix triggers research planning mode per the system prompt (task 6.2).

Sub-task parsing from Planner output: extract numbered items or markdown headers. Each sub-task becomes a synthetic Task dispatched to `researchRunner.runResearch()` sequentially.
