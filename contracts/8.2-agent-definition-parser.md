# Task 8.2 -- Agent Definition Parser

## Objective

Parse the markdown content of an `agent.md` file into a typed `AgentDefinition`, extracting all sections (Identity, Description, Schedule, Tools, Input, Output, Depth, Budget, Adapter) with validation.

## Context

Custom agents are defined as markdown files at `agents/<name>/agent.md`. NanoClaw reads these files to know what agents exist and how to run them. The parser is a pure function — it takes raw markdown and returns a well-typed `AgentDefinition` from task 8.1. It's the bridge between human-authored agent definitions and the fleet system's type-safe internals.

Depends on 8.1 (types). Blocks 8.3 (loader).

## Requirements

- [ ] `parseAgentDefinition(content: string, configPath: string): AgentDefinition` function
- [ ] Extract sections by splitting on `## ` headings
- [ ] Parse Identity section: `agent_type`, `model`, `provider` (key: value pairs)
- [ ] Parse Description section: full text as string
- [ ] Parse Schedule section: `cron: "..."` or `on_commit: <branch>` / `on_task_complete: <pattern>` / `on_gate_approved: <type>`
- [ ] Parse Tools section: YAML-like list (`- tool_name:` with optional nested config, or `- tool_name: true`)
- [ ] Parse Input section: full text as string (instructions for the agent)
- [ ] Parse Output section: list of `- destination: format` or `- destination: format (key: value)`
- [ ] Parse Depth section: `level: N` (integer 1-10)
- [ ] Parse Budget section: `max_cost_per_run: $N.NN`
- [ ] Parse Adapter section (optional): `type: service` with service config fields, or `type: llm` (default)
- [ ] Generate stable `id` from agent directory name (extracted from configPath)
- [ ] Extract agent `name` from the markdown H1 title (`# Agent Name`)
- [ ] Validation: cron must have 5 fields, depth must be 1-10, budget must be positive, provider must be a known Provider type from `src/agents/types.ts`
- [ ] Return descriptive error (throw) for invalid or unparseable definitions
- [ ] Pure function with no side effects

## Interface Contract

**Inputs:** Raw markdown string, config file path.

**Outputs:** `AgentDefinition` (from task 8.1 types).

**Side Effects:** None (pure function).

## Technical Approach

```typescript
// src/fleet/parser.ts
export function parseAgentDefinition(content: string, configPath: string): AgentDefinition {
  const sections = extractSections(content);
  const name = extractTitle(content); // H1 line
  const id = extractIdFromPath(configPath); // agents/tbpn-digest/agent.md → "tbpn-digest"
  
  const identity = parseIdentity(sections["Identity"]);
  const schedule = parseSchedule(sections["Schedule"]);
  const tools = parseTools(sections["Tools"]);
  const output = parseOutput(sections["Output"]);
  const depth = parseDepth(sections["Depth"]);
  const budget = parseBudget(sections["Budget"]);
  const adapter = parseAdapter(sections["Adapter"]); // optional, defaults to { type: "llm" }
  
  return {
    id, name, configPath,
    agentType: identity.agent_type,
    model: identity.model,
    provider: identity.provider,
    description: (sections["Description"] || "").trim(),
    schedule, tools,
    input: (sections["Input"] || "").trim(),
    outputDestinations: output,
    depth: depth ?? undefined,
    budgetPerRun: budget ?? undefined,
    adapterConfig: adapter,
    enabled: true,
  };
}
```
