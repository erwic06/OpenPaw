# Task 4.4 -- Researcher Agent System Prompt

## Objective

Create the Researcher agent system prompt with strong anti-sycophancy focus, research methodology, citation requirements, and depth awareness.

## Context

The Researcher uses Gemini 3.1 Pro, which tends toward sycophancy. The system prompt is the primary mitigation: it enforces evidence-based claims, explicit confidence levels, mandatory "insufficient evidence" statements, and adversarial self-checking. The cross-model review (Gemini researches, Claude reviews in Task 4.6) provides a second layer.

No dependencies. Content-only task.

## Requirements

- [ ] Anti-sycophancy directives:
  - Every factual claim must cite a source
  - Explicit confidence levels per section (high/medium/low)
  - Mandatory "evidence insufficient" when sources are weak or contradictory
  - Adversarial self-check: before finalizing, re-examine each claim critically
  - Never present speculation as fact
  - Prefer "I found no evidence" over fabricating plausible-sounding claims
- [ ] Research methodology:
  - Start broad, then narrow to specifics
  - Triangulate: cross-reference claims across multiple independent sources
  - Evaluate source reliability (official docs > blog posts > forum comments)
  - Check publication dates (prefer recent sources for fast-moving topics)
- [ ] Citation requirements:
  - Inline citations: [1], [2], etc.
  - Numbered source list at end with URL, title, access date
  - Every factual claim must have at least one citation
  - Sources must be real URLs (not fabricated)
- [ ] Depth awareness:
  - Prompt includes depth level (1-10)
  - Agent scopes work accordingly (depth 1-2: quick answer, depth 9-10: exhaustive)
  - Token budget guidance per depth level
- [ ] Output format:
  - Structured markdown with clear section headings
  - Confidence level stated per section
  - Source list at end
  - Executive summary at top
- [ ] Filesystem access: `research/*` read-write, `src/*` none
- [ ] Tools: BrowserUse for web browsing, file I/O for writing the research brief
- [ ] Hard stop conditions: topic requires classified sources, all sources contradictory with no resolution

## What NOT to Do

- Do not create code -- this is a content-only task
- Do not promise capabilities the Researcher doesn't have (e.g., code execution)
- Do not allow the Researcher to write to `src/*` or modify the implementation plan
- Do not make the prompt so long it consumes excessive context tokens
