# Task 3.5 -- Agent SDK Integration + LLMAdapter

## Objective

Implement `LLMAdapter` -- the core integration between NanoClaw and the Claude Agent SDK. This adapter starts headless Claude Code sessions, passes MCP tools, streams messages, tracks token usage, and returns results. It implements the `AgentAdapter` interface from Task 3.1.

## Context

The Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`) provides programmatic access to Claude Code as a headless agent. The SDK's `query()` function spawns a Claude Code subprocess, which uses MCP tools defined via `tool()` and `createSdkMcpServer()`. NanoClaw's `LLMAdapter` wraps this to manage session lifecycle, cost tracking, and cancellation.

Depends on Task 3.1 (types) and Task 3.3 (cost tracking).

## Requirements

- [ ] `LLMAdapter` implements `AgentAdapter` interface
- [ ] `trigger(input)`: create session record in SQLite, call Agent SDK `query()` with system prompt, model config, MCP server, and permission bypass
- [ ] `status(sessionId)`: return current session status from in-memory state
- [ ] `output(sessionId)`: return collected result text, artifacts, usage, and cost
- [ ] `cancel(sessionId)`: abort the session via AbortController
- [ ] Iterate the `query()` async generator, processing:
  - `SDKAssistantMessage` -- extract `usage` for cost tracking, update last activity timestamp
  - `SDKResultMessage` -- extract final usage, terminal state
  - Error messages -- detect rate limits for fallback routing (Task 3.7)
- [ ] Pass `permissionMode: 'bypassPermissions'` for headless operation
- [ ] Pass secrets via `env` option (never set in `process.env`)
- [ ] Store AbortController per session for cancellation
- [ ] Claude Code CLI must be available inside Docker container
- [ ] DI pattern for testability
- [ ] Unit tests with mocked Agent SDK

## Interface Contract

**Inputs:** `AgentInput` (task context, system prompt, model config, MCP server config, budget).

**Outputs:** `AgentOutput` (result text, artifacts, usage, cost, terminal state) via `output()`.

**Side Effects:** Spawns Claude Code subprocess. Writes to SQLite sessions table. Calls cost tracking module.

The `query()` options include: `prompt`, `systemPrompt`, `model`, `permissionMode`, `cwd`, `maxTurns`, `abortController`, `env`. The `cwd` option sets the workspace directory for the Claude Code session (Task 3.9 configures this to point at the per-session workspace from Task 3.4).

Key design decision: `trigger()` starts the session asynchronously and returns immediately with the session ID. The session runs in the background. The caller polls via `status()` or awaits completion.
