# Task 7.3 -- Task Creation and Action Endpoints

## Objective

Implement the write/action REST endpoints for creating tasks and projects from the web dashboard, and wire all API routes into the NanoClaw startup sequence.

## Context

Tasks 7.1 (router) and 7.2 (read endpoints) provide the routing foundation and data queries. This task adds the action endpoints that trigger work: creating research tasks, coding tasks, and projects. These endpoints mirror the Telegram intake flows from Phase 6 but accept JSON payloads from the web frontend.

Depends on 7.2. Phase 6 intake modules may not be implemented yet — if so, the endpoints return 501 until the backing flows are wired.

## Requirements

- [ ] `POST /api/tasks` -- create a task; body: `{ type: "research" | "coding", prompt: string, depth?: number, projectName?: string }`
- [ ] `POST /api/projects` -- create a project; body: `{ name: string, description: string }`
- [ ] `GET /api/tasks/:id` -- task detail (reads from implementation plan via plan parser)
- [ ] `GET /api/tasks/:id/sessions` -- session history for task (queries sessions table by task_id)
- [ ] Extend `ApiDeps` with optional intake function references (research, coding, project flows)
- [ ] When intake functions are not provided (Phase 6 not yet wired), POST endpoints return `{ error: "Not yet available" }` with status 501
- [ ] When intake functions are provided, POST endpoints call them and return `{ data: { taskId, status: "accepted" } }`
- [ ] Validation: research requires non-empty prompt and optional depth 1-10; coding requires projectName and prompt; project requires name
- [ ] 400 response for invalid request body with descriptive error message
- [ ] Wire all route modules (7.2 + 7.3) into a unified route array via `allRoutes(deps)`

## Interface Contract

**Inputs:** JSON request bodies from web dashboard. ApiDeps with optional intake functions.

**Outputs:** JSON responses with task ID and status. Unified route array for registration.

**Side Effects:** POST endpoints trigger background task execution (research runner, session runner). POST /api/projects creates DB entry.

## Technical Approach

```typescript
// src/api/routes/tasks.ts
export function taskRoutes(deps: ApiDeps): Route[] {
  return [
    {
      method: "POST",
      pattern: "/api/tasks",
      handler: async (ctx) => {
        const { type, prompt, depth, projectName } = ctx.body as any;
        if (!prompt?.trim()) return errorResponse("prompt is required", 400);
        
        if (type === "research") {
          if (!deps.createResearchTask) return errorResponse("Not yet available", 501);
          const taskId = await deps.createResearchTask({ prompt, depth: depth ?? 5 });
          return jsonResponse({ data: { taskId, status: "accepted" } });
        }
        // ...
      },
    },
  ];
}
```
