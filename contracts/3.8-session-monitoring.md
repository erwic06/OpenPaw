# Task 3.8 -- Session Monitoring

## Objective

Implement heartbeat-based session monitoring that detects hung or crashed sessions. If a session produces no output for 10 minutes, kill it, mark the task as FAILED, and notify the human via Telegram.

## Context

The design doc (Section 4, F2) specifies: if no tool call or text output for 10 minutes, session is hung. If API connection drops, session is crashed. The monitor checks active sessions periodically and acts on stale ones. Depends on Task 3.5 (LLMAdapter exposes last-activity timestamps).

## Requirements

- [ ] `SessionMonitor` tracks active sessions with their last activity timestamp
- [ ] `recordActivity(sessionId)` -- called by LLMAdapter on every message from the Agent SDK
- [ ] `startMonitoring(sessionId)` / `stopMonitoring(sessionId)` lifecycle methods
- [ ] Periodic check (every 60 seconds): if `now - lastActivity > 10 minutes`, session is hung
- [ ] On hung detection: cancel session (abort), update SQLite (FAILED, error: "Session hung -- no output for 10 minutes"), send Telegram alert
- [ ] On API connection drop: LLMAdapter reports this; monitor handles cleanup
- [ ] DI pattern: `MonitorDeps = { cancelSession, db, sendAlert }`
- [ ] Unit tests with fake timers

## Interface Contract

**Inputs:** Session activity updates from LLMAdapter.

**Outputs:** Hung/crash events trigger side effects (cancel, DB update, alert).

**Side Effects:** Kills hung sessions, updates SQLite, sends Telegram alerts.

## Technical Approach

```typescript
class SessionMonitor {
  private sessions = new Map<string, number>(); // sessionId -> lastActivityMs
  private interval: Timer | null = null;

  start() { this.interval = setInterval(() => this.check(), 60_000); }
  stop() { clearInterval(this.interval); }

  recordActivity(sessionId: string) {
    this.sessions.set(sessionId, Date.now());
  }

  private check() {
    const now = Date.now();
    for (const [id, lastActive] of this.sessions) {
      if (now - lastActive > 10 * 60 * 1000) {
        this.handleHung(id);
      }
    }
  }
}
```

The LLMAdapter calls `recordActivity(sessionId)` every time it processes an `SDKAssistantMessage` or `SDKUserMessage` from the `query()` generator.
