# Task 6.7 -- Coding Intake Flow

## Objective

Implement the `/coding` intake flow that validates an existing project, adds a new task to its implementation plan, and dispatches it via the existing `SessionRunner`.

## Context

The design doc (lines 1151-1155, 1656-1658) specifies: `/coding <project-name> <prompt>` adds a task to an existing project's `implementation_plan.md` and dispatches it through the normal task flow. Unlike `/research` and `/project`, coding tasks are always tied to an existing project.

Projects created by `/project` (task 6.6) live at `/workspaces/<project-name>/`. This task validates the project exists, generates the next task ID, appends a task entry to the plan, and enqueues it on the SessionRunner.

Depends on 6.1 (parser types), 6.4 (dispatch). Blocks 6.9.

## Requirements

- [ ] `CodingIntakeDeps` interface with DI:
  - `sendMessage: (chatId: number | string, text: string, options?: { parseMode?: string }) => Promise<void>`
  - `chatId: number`
  - `runner: SessionRunner`
  - `workspacesDir: string` (base dir, e.g., `/workspaces`)
  - `appendTaskFn?: (planPath: string, task: Task) => Promise<void>` -- override for testing
- [ ] `handleCoding(deps, projectName, prompt): Promise<void>`
- [ ] `generateNextTaskId(planPath: string): Promise<string>` -- reads plan, finds highest task ID, increments
- [ ] `appendTaskToPlanner(planPath: string, task: Task): Promise<void>` -- appends task entry to plan file
- [ ] `/coding` flow:
  1. Validate project exists at `{workspacesDir}/{projectName}/`
  2. Validate `implementation_plan.md` exists in project dir
  3. Generate next task ID (e.g., if last task is `3.4`, next is `3.5`)
  4. Create Task object with generated ID, prompt as title, status `ready`
  5. Append task entry to implementation_plan.md
  6. Enqueue task via `runner.enqueue(task)`
  7. Send confirmation to Telegram: "Task {id} added to {projectName}"
  8. On validation failure: send error ("Project '{name}' not found")
- [ ] Exported from `src/intake/index.ts` barrel

## Interface Contract

**Inputs:** Project name and prompt from `/coding` command.

**Outputs:** Task appended to project's implementation_plan.md, enqueued on SessionRunner.

**Side Effects:** Modifies project's implementation_plan.md file, enqueues task for execution.

## Technical Approach

```typescript
// src/intake/coding.ts
import { existsSync } from "fs";
import { join } from "path";
import type { Task } from "../plan/types.ts";
import type { SessionRunner } from "../agents/runner.ts";
import { parsePlan } from "../plan/parser.ts";

export async function handleCoding(
  deps: CodingIntakeDeps,
  projectName: string,
  prompt: string,
): Promise<void> {
  const projectDir = join(deps.workspacesDir, projectName);
  const planPath = join(projectDir, "implementation_plan.md");

  if (!existsSync(projectDir)) {
    await deps.sendMessage(deps.chatId, `Project '${projectName}' not found.`);
    return;
  }

  if (!existsSync(planPath)) {
    await deps.sendMessage(deps.chatId, `No implementation plan found for '${projectName}'.`);
    return;
  }

  const taskId = await generateNextTaskId(planPath);
  const task: Task = {
    id: taskId,
    title: prompt.length > 120 ? prompt.slice(0, 120) + "..." : prompt,
    status: "ready",
    type: "code",
    contract: "",
    dependencies: [],
    assigned: "intake",
    artifacts: [],
    acceptance: prompt,
    notes: [],
  };

  const appendFn = deps.appendTaskFn ?? appendTaskToPlan;
  await appendFn(planPath, task);
  await deps.runner.enqueue(task);
  await deps.sendMessage(deps.chatId, `Task ${taskId} added to ${projectName}.`);
}
```

Task ID generation: parse the plan with `parsePlan()` from `src/plan/parser.ts`, find the highest numeric task ID (e.g., `3.4`), increment the minor version (→ `3.5`). If no tasks exist, start at `1.1`.

The `appendTaskToPlan` function writes the task entry in the standard format (matching the implementation_plan.md structure) before the Session Log table.
