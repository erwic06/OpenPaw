# Task 8.3 -- Agent Definition Loader and File Watcher

## Objective

Scan the `agents/` directory for subdirectories containing `agent.md`, parse each definition, sync to the `agent_definitions` database table, and watch for changes with automatic hot-reload.

## Context

Agent definitions live as markdown files in the repo. NanoClaw needs to discover them on startup and react to changes. The file watcher follows the same pattern as `src/plan/reader.ts` (fs.watch + polling fallback). On change, definitions are re-parsed and the database is synced: new definitions are inserted, changed definitions are updated, and removed definitions are soft-disabled.

Depends on 8.1 (types/DB) and 8.2 (parser). Blocks 8.5 (scheduler) and 8.6 (events).

## Requirements

- [ ] `loadAllAgentDefinitions(agentsDir: string): AgentDefinition[]` -- scan directory, parse all agent.md files
- [ ] `syncDefinitionsToDb(db: Database, definitions: AgentDefinition[]): void` -- upsert definitions, soft-disable missing ones
- [ ] `watchAgentDefinitions(agentsDir: string, onChange: (defs: AgentDefinition[]) => void): { stop: () => void }` -- file watcher
- [ ] Directory scanning: find all `agents/*/agent.md` files (one level deep)
- [ ] Skip system prompt directories (e.g., `agents/coder/`, `agents/researcher/`) that don't have `agent.md`
- [ ] Parse errors logged and skipped (don't crash on one bad definition)
- [ ] `syncDefinitionsToDb`:
  - Insert new definitions not in DB
  - Update existing definitions that changed (compare configPath content hash)
  - Set `enabled = false` for definitions in DB but no longer on disk (soft delete)
  - Compute `next_run_at` for cron-type definitions
- [ ] File watcher: `fs.watch` on agents directory with 500ms debounce, polling fallback at 5min intervals
- [ ] `onChange` callback receives the full updated list of enabled definitions
- [ ] `{ stop }` return for clean shutdown

## Interface Contract

**Inputs:** Directory path for agents, Database instance.

**Outputs:** `loadAllAgentDefinitions` function, `syncDefinitionsToDb` function, `watchAgentDefinitions` function.

**Side Effects:** Reads filesystem. Writes to agent_definitions table. fs.watch file system watcher.

## Technical Approach

```typescript
// src/fleet/loader.ts
export function loadAllAgentDefinitions(agentsDir: string): AgentDefinition[] {
  const defs: AgentDefinition[] = [];
  const entries = readdirSync(agentsDir, { withFileTypes: true });
  for (const entry of entries) {
    if (!entry.isDirectory()) continue;
    const agentMdPath = join(agentsDir, entry.name, "agent.md");
    if (!existsSync(agentMdPath)) continue;
    try {
      const content = readFileSync(agentMdPath, "utf-8");
      defs.push(parseAgentDefinition(content, agentMdPath));
    } catch (err) {
      console.error(`[fleet] failed to parse ${agentMdPath}: ${err}`);
    }
  }
  return defs;
}
```
