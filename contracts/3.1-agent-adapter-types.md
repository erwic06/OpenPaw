# Task 3.1 -- AgentAdapter Interface and Types

## Objective

Define the `AgentAdapter` interface and all associated types that serve as the contract between NanoClaw's orchestration layer and its agent implementations (LLMAdapter in Phase 3, ServiceAdapter in Phase 8).

## Context

The design document (Section 1) specifies a uniform `AgentAdapter` interface with four methods: trigger, status, output, cancel. Phase 3 introduces the `LLMAdapter` implementation; Phase 8 adds `ServiceAdapter`. Defining the interface and types now costs nothing and prevents refactoring later. This task also defines the tiered model roster (Heavy/Standard/Light) that maps agent types to primary + fallback model configurations.

No dependencies. Pure types module.

## Requirements

- [ ] `AgentAdapter` interface with `trigger`, `status`, `output`, `cancel` methods matching design doc signatures
- [ ] `AgentStatus` type: `"running" | "complete" | "failed" | "waiting_hitl"`
- [ ] `AgentInput` type with fields for task context, system prompt path, model config, tools, and budget
- [ ] `AgentOutput` type with session results, artifacts, usage, cost, and terminal state
- [ ] `ModelTier` type: `"heavy" | "standard" | "light"`
- [ ] `ModelConfig` type mapping a tier to primary model/provider and fallback model/provider
- [ ] `ModelRoster` as the full roster configuration (all three tiers)
- [ ] `DEFAULT_ROSTER` constant with the decided model assignments:
  - Heavy: Opus 4.6 / GPT-5.4 High
  - Standard: Sonnet 4.6 / GPT-5.4 Medium
  - Light: Haiku 4.5 / GPT-5.4 Mini
- [ ] All types exported from barrel `src/agents/index.ts`

## Interface Contract

**Inputs:** None (pure types module).

**Outputs:** TypeScript types and interfaces importable by other modules.

**Side Effects:** None.

## Technical Approach

Define all types in `src/agents/types.ts`. Create barrel export in `src/agents/index.ts`. The `AgentAdapter` interface uses the exact method signatures from the design document. Model IDs should use the canonical strings for each provider (e.g., `"claude-opus-4-6"`, `"claude-sonnet-4-6"`, `"claude-haiku-4-5"`, `"gpt-5.4"`, `"gpt-5.4-mini"`). Verify the correct model ID strings during implementation.

## Files to Create/Modify

- `src/agents/types.ts` -- All type definitions
- `src/agents/index.ts` -- Barrel export

## Acceptance Criteria

- [ ] `bun run typecheck` passes with no errors
- [ ] `AgentAdapter` interface has exactly 4 methods: `trigger`, `status`, `output`, `cancel`
- [ ] `AgentStatus` is a union of exactly 4 string literals
- [ ] `ModelTier` has 3 tiers, `ModelRoster` maps all 3 to `ModelConfig`
- [ ] `DEFAULT_ROSTER` constant is defined with the correct model assignments
- [ ] All types exported from `src/agents/index.ts`

## What NOT to Do

- Do not implement any adapter classes -- just the interface and types
- Do not add any runtime dependencies -- this is a pure types module
- Do not define tool types here -- those belong in Task 3.6

## Recovery Guidance

- If model ID strings are wrong: check provider documentation for canonical model identifiers
- If typecheck fails: check that all referenced types are defined before use
