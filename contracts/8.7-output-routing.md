# Task 8.7 -- Output Routing Engine

## Objective

Implement the output routing engine that delivers agent run results to configured destinations: Telegram (formatted message), GitHub (commit to repo), and webapp (always-on via existing data).

## Context

After an agent run completes, its output needs to be delivered to the destinations specified in the agent definition's Output section. Each destination has its own handler: Telegram sends a formatted message, GitHub commits a file to the repo. Notion is deferred — stub only.

Depends on 8.1 (types, OutputDestination). Independent of scheduling (8.5-8.6).

## Requirements

- [ ] `routeOutput(deps: RoutingDeps, agentDef: AgentDefinition, output: AgentOutput): Promise<RoutingResult[]>`
- [ ] `RoutingDeps`: `{ sendMessage: SendMessageFn; chatId: string; repoDir: string; spawnFn?: typeof Bun.spawn }`
- [ ] `RoutingResult`: `{ destination: string; success: boolean; error?: string }`
- [ ] Telegram routing:
  - "summary" format: send condensed output via `sendMessage` with HTML formatting
  - "full_report" format: send truncated preview with note that full report is on GitHub
  - Message includes: agent name, run timestamp, status, summary text
- [ ] GitHub routing:
  - Commit output file at specified path (e.g., `research/tbpn/2026-04-02.md`)
  - Git operations: write file → `git add` → `git commit` → `git push`
  - Commit message: `[agent] <agent-name>: output for <date>`
  - Use `Bun.spawn` for git commands (same pattern as workspace manager)
- [ ] Notion routing: stub that logs "Notion routing deferred" and returns `{ success: false, error: "deferred" }`
- [ ] Webapp routing: no-op (data is in agent_runs/sessions tables, API serves it)
- [ ] Returns `RoutingResult[]` -- one entry per destination, serialized to JSON for `agent_runs.output_routed_to`
- [ ] Individual destination failures don't block other destinations
- [ ] DI via `spawnFn` for testing git operations without real git

## Interface Contract

**Inputs:** AgentDefinition (output destinations), AgentOutput (session results).

**Outputs:** `RoutingResult[]` indicating per-destination success/failure.

**Side Effects:** Telegram messages sent. Git commits pushed to repo.

## Technical Approach

```typescript
// src/fleet/router.ts
export async function routeOutput(
  deps: RoutingDeps,
  agentDef: AgentDefinition,
  output: AgentOutput,
): Promise<RoutingResult[]> {
  const results: RoutingResult[] = [];
  for (const dest of agentDef.outputDestinations) {
    try {
      switch (dest.type) {
        case "telegram":
          await routeToTelegram(deps, agentDef, output, dest);
          results.push({ destination: "telegram", success: true });
          break;
        case "github":
          await routeToGithub(deps, agentDef, output, dest);
          results.push({ destination: `github:${dest.path}`, success: true });
          break;
        // ...
      }
    } catch (err) {
      results.push({ destination: dest.type, success: false, error: String(err) });
    }
  }
  return results;
}
```
