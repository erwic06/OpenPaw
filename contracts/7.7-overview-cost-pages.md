# Task 7.7 -- Overview and Cost Dashboard Pages

## Objective

Build the Overview tab (landing page) with daily spend display, summary cards, and a chronological activity feed with actionable gate items.

## Context

The Overview tab is the first thing the user sees. It shows at-a-glance system health: daily spend vs budget, active sessions, pending gates, and recent activity. Pending gates in the activity feed are actionable — the user can approve/deny inline.

Depends on 7.6 (Next.js scaffold with layout shell and API/WS clients) and 7.2 (REST endpoints for sessions, gates, costs).

## Requirements

- [ ] Overview page at `web/src/app/overview/page.tsx`
- [ ] Spend bar component: daily spend vs budget with progress bar (e.g., "$4.20 / $50.00 · 8.4%")
- [ ] Summary cards (4): Active Sessions, Pending Gates, Research Tasks, Automations Running
- [ ] Each card shows a count and status indicator
- [ ] Activity feed: chronological list of recent events (session started/completed/failed, gate pending/resolved, research complete)
- [ ] Activity feed items show: timestamp, description, agent/source, cost (if applicable)
- [ ] Pending gate items in feed are actionable: [Approve] [Deny] buttons
- [ ] Approving/denying a gate calls `POST /api/gates/:id/decide` and updates the feed
- [ ] Auto-refresh via notification WebSocket (new events appear without page reload)
- [ ] Data fetched from: `GET /api/sessions`, `GET /api/gates/pending`, `GET /api/cost/daily`
- [ ] Loading state while data is fetched
- [ ] Empty state when no recent activity

## Interface Contract

**Inputs:** REST API responses (sessions, gates, costs). WebSocket notification events.

**Outputs:** React components rendering the Overview tab.

**Side Effects:** Gate approval/denial via POST.

## Technical Approach

Use React `useEffect` for initial data fetch and WebSocket subscription. The activity feed merges sessions and gates into a unified chronological list, sorted by timestamp descending. Gate actions use optimistic UI updates — show the result immediately, revert on error.

## Files to Create/Modify

## Acceptance Criteria

- [ ] `cd web && npm run build` succeeds
- [ ] Overview page renders with spend bar, summary cards, and activity feed
- [ ] Spend bar shows daily spend as progress bar with percentage
- [ ] Summary cards show correct counts (or 0 when no data)
- [ ] Activity feed displays recent events in chronological order
- [ ] Gate items in feed have Approve/Deny buttons
- [ ] Clicking Approve calls POST /api/gates/:id/decide with decision "approved"
- [ ] Loading state shown while data fetches
- [ ] Empty state shown when no activity

## What NOT to Do

- Do not implement the command input bar's submission logic -- it's a UI element in the header from 7.6, the dialog is in 7.9
- Do not fetch agent/automation data -- that depends on Phase 8; show 0 or placeholder
- Do not add pagination to the activity feed -- limit to last 50 items for now

## Recovery Guidance

- If API is not running, show error state with retry button
- Test with mock data first, then wire to real API
