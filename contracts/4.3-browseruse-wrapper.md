# Task 4.3 -- BrowserUse Tool Wrapper

## Objective

Implement an adapter-agnostic BrowserUse tool wrapper for Cloud-mode browser automation. Returns structured results that any agent adapter can consume.

## Context

BrowserUse provides browser automation via Cloud API. The wrapper is used by the Researcher agent (via Gemini function calling) for web browsing during research. Cloud-only mode avoids bundling Chromium in the Docker container.

No dependencies. Can be built in parallel with other Phase 4 tasks.

## Requirements

- [ ] `BrowserUseMode` type: `"cloud"` (CLI mode deferred to future task)
- [ ] `BrowserUseDeps` interface: `{ cloudApiKey, fetchFn? }` (DI for testing)
- [ ] `BrowserUseResult` type: `{ url, title, content, error? }`
- [ ] `browseUrl(deps, url, options?)` -- make Cloud API call, return structured result
- [ ] `getBrowserUseToolDeclaration()` -- returns Gemini-compatible function declaration
- [ ] Error handling: API failures return result with error field (don't throw)
- [ ] Content truncation: limit returned content to prevent token explosion
- [ ] Unit tests with mocked HTTP (fetchFn DI)

## Interface Contract

**Inputs:** URL string, optional action/instructions.

**Outputs:** `BrowserUseResult` with page content and metadata.

**Side Effects:** HTTP calls to BrowserUse Cloud API.
