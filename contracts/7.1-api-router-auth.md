# Task 7.1 -- API Router and Auth Middleware

## Objective

Build the HTTP routing foundation for the NanoClaw API module, replacing the bare `Bun.serve` health endpoint with a proper URL-pattern router and Cloudflare Access JWT validation middleware.

## Context

Phase 7 adds a REST+WebSocket API to NanoClaw so the web dashboard can interact with the system. The API router is the foundation — all subsequent API tasks (7.2-7.4, 7.10) depend on it. The current `Bun.serve` in `src/index.ts` (line 182) only returns "ok" for any request. This task upgrades it to a full router with URL pattern matching, method dispatch, and auth.

Auth is handled by Cloudflare Access — the middleware validates the `cf-access-jwt-assertion` header using CF's public keys. No custom auth logic beyond JWT validation.

## Interface Contract

**Inputs:** HTTP requests via `Bun.serve({ fetch })`.

**Outputs:** `createRouter` function, `validateCfAccess` function, `ApiDeps`/`ApiContext`/`Route` types. All exported from `src/api/index.ts`.

**Side Effects:** None (router is a pure function factory; auth fetches CF certs on first request, then caches).

## Technical Approach

The router uses simple string pattern matching — split URL path and pattern on `/`, match segments, extract `:param` values. No regex or URLPattern API needed for the limited set of API routes.

```typescript
// src/api/router.ts
export function createRouter(
  routes: Route[],
  deps: ApiDeps,
): (req: Request) => Response | Promise<Response> {
  return async (req: Request) => {
    const url = new URL(req.url);
    const method = req.method;
    
    for (const route of routes) {
      const params = matchPattern(route.pattern, url.pathname);
      if (params !== null && route.method === method) {
        const body = hasBody(method) ? await req.json().catch(() => null) : null;
        const ctx: ApiContext = { params, query: url.searchParams, body, deps };
        return route.handler(ctx);
      }
    }
    
    // Check if path matches any route with different method
    const pathMatched = routes.some(r => matchPattern(r.pattern, url.pathname) !== null);
    if (pathMatched) return errorResponse("Method not allowed", 405);
    return errorResponse("Not found", 404);
  };
}
```

For CF Access JWT validation, use Bun's built-in `crypto.subtle.importKey` and `crypto.subtle.verify` for RS256 JWT verification. Parse the JWT header to find the `kid`, look up the matching key from CF's JWKS endpoint, verify signature and `aud` claim.
