# Task 6.9 -- Docker Wiring and End-to-End Verification

## Objective

Wire all Phase 6 intake modules into `src/index.ts`, add the `github_token` Docker secret, and verify end-to-end command parsing and dispatch.

## Context

This is the integration task for Phase 6 (same pattern as 4.8 and 5.6). All intake modules (6.1-6.8) are composed in `src/index.ts` via `initCommandDispatch()` with handler callbacks that route to the intake flow functions.

Depends on 6.5, 6.6, 6.7, 6.8 (all intake flows). Final task in Phase 6.

## Requirements

- [ ] **docker-compose.yml:** Add `github_token` secret:
  ```yaml
  github_token:
    file: ./secrets/github_token
  ```
- [ ] **src/secrets.ts:** Load `github_token` from secrets volume (same pattern as existing secrets)
- [ ] **src/index.ts:** Wire intake dispatch:
  - Import `initCommandDispatch` from `src/intake/index.ts`
  - Import handler functions: `handleResearch`, `handleDeepResearch`, `handleProject`, `handleCoding`, `handleStatus`, `handleAgents`
  - Construct deps for each handler from existing initialized modules (db, sendMessage, chatId, researchRunner, runner, etc.)
  - Call `initCommandDispatch({ onMessage, sendMessage, chatId, onResearch: ..., onDeepResearch: ..., ... })`
  - Register after `initGates()` (gates should register their handler first)
- [ ] **Graceful degradation:**
  - If `github_token` missing: `/project` sends "GitHub token not configured" error
  - If `gemini_api_key` missing: `/research` and `/dr` send "Research runner not available" error
  - All other commands work regardless of optional secrets
- [ ] **SIGTERM handler:** No changes needed (intake is stateless, no cleanup required)
- [ ] Docker build succeeds with all new modules

## Interface Contract

**Inputs:** All Phase 6 modules assembled.

**Outputs:** Fully wired command dispatch in `src/index.ts`.

**Side Effects:** Modifies startup sequence. Adds Docker secret.

## Technical Approach

```typescript
// In src/index.ts, after initGates() and after researchRunner/runner initialization:

import { initCommandDispatch } from "./intake/index.ts";
import {
  handleResearch,
  handleDeepResearch,
} from "./intake/research.ts";
import { handleProject } from "./intake/project.ts";
import { handleCoding } from "./intake/coding.ts";
import { handleStatus, handleAgents } from "./intake/status.ts";

// ... after runner initialization:

const researchIntakeDeps = {
  db,
  sendMessage,
  chatId: parseInt(chatId),
  researchRunner,
  anthropicApiKey,
  plannerPromptPath: join(__dirname, "../agents/planner/system_prompt.md"),
};

const projectIntakeDeps = {
  db,
  sendMessage,
  chatId: parseInt(chatId),
  githubDeps: githubToken ? { githubToken } : null,
  anthropicApiKey,
  plannerPromptPath: join(__dirname, "../agents/planner/system_prompt.md"),
  workspacesDir: "/workspaces",
};

initCommandDispatch({
  onMessage,
  sendMessage,
  chatId: parseInt(chatId),
  onResearch: async (cmd) => {
    if (!researchRunner) {
      await sendMessage(chatId, "Research runner not available (missing gemini_api_key).");
      return;
    }
    await handleResearch(researchIntakeDeps, cmd.depth, cmd.prompt);
  },
  onDeepResearch: async (cmd) => {
    if (!researchRunner) {
      await sendMessage(chatId, "Research runner not available (missing gemini_api_key).");
      return;
    }
    await handleDeepResearch(researchIntakeDeps, cmd.prompt);
  },
  onProject: async (cmd) => {
    if (!githubToken) {
      await sendMessage(chatId, "GitHub token not configured.");
      return;
    }
    await handleProject(projectIntakeDeps, cmd.prompt);
  },
  onCoding: async (cmd) => {
    if (!runner) {
      await sendMessage(chatId, "Session runner not available.");
      return;
    }
    await handleCoding(codingIntakeDeps, cmd.projectName, cmd.prompt);
  },
  onStatus: async () => await handleStatus(statusDeps),
  onAgents: async () => await handleAgents(statusDeps),
});
```

Registration order in `index.ts`:
1. `initGates(gateDeps)` -- gates register their handler first
2. `initCommandDispatch(dispatchDeps)` -- command dispatch registered second

Both handlers are called for every message. Gates skip `/` messages. Dispatch skips non-`/` messages. No conflict.
