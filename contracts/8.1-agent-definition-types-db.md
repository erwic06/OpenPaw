# Task 8.1 -- Agent Definition Types and DB Schema

## Objective

Define the TypeScript types for the fleet system (agent definitions, runs, scheduling, output routing, adapter config) and add the `agent_definitions` and `agent_runs` tables to SQLite with CRUD functions.

## Context

Phase 8 adds custom agent scheduling and management. This task establishes the data foundation — types and database schema — that all subsequent fleet tasks depend on. The types model everything an `agent.md` definition file can express. The DB functions follow the existing prepared-statement pattern in `src/db/index.ts`.

No dependencies. Blocks 8.2 (parser), 8.3 (loader), 8.4 (ServiceAdapter), 8.5 (scheduler), 8.7 (routing).

## Requirements

- [ ] `AgentDefinition` interface: id, name, configPath, agentType, model, provider, description, schedule (ScheduleConfig), tools, input, outputDestinations, depth, budgetPerRun, adapterConfig, enabled
- [ ] `AgentRun` interface: id, agentId, sessionId, triggeredBy, triggerDetail, startedAt, endedAt, status, outputRoutedTo
- [ ] `ScheduleType` union: `"cron" | "event" | "manual"`
- [ ] `ScheduleConfig`: `{ type: ScheduleType; expression?: string }` (expression is cron or event spec)
- [ ] `OutputDestination` discriminated union:
  - `{ type: "telegram"; format: "summary" | "full_report" }`
  - `{ type: "github"; format: "summary" | "full_report"; path: string }`
  - `{ type: "webapp" }`
- [ ] `AdapterConfig` discriminated union:
  - `{ type: "llm" }` (default)
  - `{ type: "service"; baseUrl: string; auth: string; healthCheck: string; triggerEndpoint: string; statusEndpoint: string; outputEndpoint: string }`
- [ ] `AgentDefinitionRow` and `AgentRunRow` interfaces in `src/db/types.ts`
- [ ] `agent_definitions` table in `src/db/schema.sql` matching design doc schema
- [ ] `agent_runs` table in `src/db/schema.sql` matching design doc schema
- [ ] DB CRUD functions: `insertAgentDefinition`, `updateAgentDefinition`, `getAgentDefinition`, `getAllAgentDefinitions`, `getEnabledAgentDefinitions`, `insertAgentRun`, `updateAgentRun`, `getAgentRunsByAgent(agentId, limit?)`, `getLatestAgentRun(agentId)`
- [ ] Barrel export from `src/fleet/index.ts`

## Interface Contract

**Inputs:** None (foundational types and schema).

**Outputs:** All fleet types exported from `src/fleet/index.ts`. DB row types in `src/db/types.ts`. CRUD functions in `src/db/index.ts`.

**Side Effects:** SQLite schema extended with 2 new tables (CREATE TABLE IF NOT EXISTS).

## Technical Approach

Follow the exact pattern from existing DB code:

```typescript
// src/db/index.ts additions
export function insertAgentDefinition(db: Database, def: Omit<AgentDefinitionRow, "created_at">): void {
  const stmt = db.prepare(`INSERT INTO agent_definitions (id, name, config_path, schedule_type, schedule_expression, enabled, next_run_at, created_at) VALUES (?, ?, ?, ?, ?, ?, ?, ?)`);
  stmt.run(def.id, def.name, def.config_path, def.schedule_type, def.schedule_expression, def.enabled ? 1 : 0, def.next_run_at, new Date().toISOString());
}
```
