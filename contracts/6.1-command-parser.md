# Task 6.1 -- Command Parser and Types

## Objective

Create a discriminated union type for all 6 Telegram intake commands and a parser that extracts structured data from raw message text, with validation for required fields and depth ranges.

## Context

Phase 6 (Telegram Intake) needs to accept commands via Telegram. The command parser is the entry point for all intake flows -- every downstream task (6.4-6.9) depends on this module's types. The design doc (Section 8, lines 1118-1155) defines 6 commands: /research, /dr, /project, /coding, /status, /agents.

No dependencies. Blocks 6.4 (dispatch), which blocks 6.5-6.8.

## Requirements

- [ ] `CommandName` union type: `"research" | "dr" | "project" | "coding" | "status" | "agents"`
- [ ] `ParsedCommand` discriminated union keyed on `command` field with 6 variants:
  - `{ command: "research"; depth: number; prompt: string }`
  - `{ command: "dr"; prompt: string }`
  - `{ command: "project"; prompt: string }`
  - `{ command: "coding"; projectName: string; prompt: string }`
  - `{ command: "status" }`
  - `{ command: "agents" }`
- [ ] `parseCommand(text: string): ParsedCommand | null` function
- [ ] `/research` validation: depth must be integer 1-10, prompt must be non-empty
- [ ] `/dr` validation: prompt must be non-empty
- [ ] `/project` validation: prompt must be non-empty
- [ ] `/coding` validation: projectName must be non-empty (first arg), prompt must be non-empty (rest)
- [ ] `/status` and `/agents` take no arguments
- [ ] Returns `null` for unrecognized commands or messages not starting with `/`
- [ ] Barrel export from `src/intake/index.ts`

## Interface Contract

**Inputs:** Raw message text string from Telegram `onMessage` handler.

**Outputs:** `ParsedCommand | null`. All types exported from `src/intake/index.ts`.

**Side Effects:** None (pure function).

## Technical Approach

```typescript
// src/intake/types.ts
export type CommandName = "research" | "dr" | "project" | "coding" | "status" | "agents";

export type ParsedCommand =
  | { command: "research"; depth: number; prompt: string }
  | { command: "dr"; prompt: string }
  | { command: "project"; prompt: string }
  | { command: "coding"; projectName: string; prompt: string }
  | { command: "status" }
  | { command: "agents" };

// src/intake/parser.ts
export function parseCommand(text: string): ParsedCommand | null {
  const trimmed = text.trim();
  if (!trimmed.startsWith("/")) return null;

  const parts = trimmed.split(/\s+/);
  const cmd = parts[0].toLowerCase().replace("/", "");
  const args = parts.slice(1);

  switch (cmd) {
    case "research": {
      const depth = parseInt(args[0], 10);
      if (isNaN(depth) || depth < 1 || depth > 10) return null;
      const prompt = args.slice(1).join(" ").trim();
      if (!prompt) return null;
      return { command: "research", depth, prompt };
    }
    case "dr":
    case "deep-research": {
      const prompt = args.join(" ").trim();
      if (!prompt) return null;
      return { command: "dr", prompt };
    }
    // ... etc
  }
}

// src/intake/index.ts
export * from "./types.ts";
export { parseCommand } from "./parser.ts";
```

Depth validation rejects non-integers (e.g., 3.5, "abc") and values outside 1-10. The `/deep-research` alias maps to the `dr` command variant.
