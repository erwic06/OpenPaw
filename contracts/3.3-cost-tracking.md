# Task 3.3 -- Cost Tracking Module

## Objective

Implement a cost tracking module that calculates USD costs from token usage based on per-model pricing, logs entries to the `cost_log` SQLite table, and provides session and daily cost aggregation.

## Context

NanoClaw needs to track costs for every API call made during headless sessions. The `cost_log` table already exists (Task 2.4). This module adds the business logic: per-model pricing lookup, cost calculation, logging, and aggregation. Used by the LLMAdapter (Task 3.5) after every API response, and by the session runner (Task 3.9) for spend threshold checks.

No dependencies (Tier 0).

## Requirements

- [ ] Hardcoded pricing for all 6 models in the roster (per 1M tokens, input and output):
  - Opus 4.6 (Anthropic)
  - Sonnet 4.6 (Anthropic)
  - Haiku 4.5 (Anthropic)
  - GPT-5.4 High (OpenAI)
  - GPT-5.4 Medium (OpenAI)
  - GPT-5.4 Mini (OpenAI)
- [ ] `logUsage(sessionId, model, provider, inputTokens, outputTokens)` -- calculate cost, insert into `cost_log`, return the calculated amount
- [ ] `getSessionCost(sessionId)` -- sum all cost_log entries for a session
- [ ] `getDailySpend(date?)` -- sum all cost_log entries for a given date (defaults to today)
- [ ] DI pattern: `CostTrackerDeps = { db: Database }` for testability
- [ ] Unit tests covering cost calculation, logging, and aggregation

## Interface Contract

**Inputs:** Token usage data (model, provider, input/output token counts).

**Outputs:**
- `logUsage(...)` -- logs cost entry to SQLite, returns calculated USD amount
- `getSessionCost(sessionId)` -- returns total USD for a session
- `getDailySpend(date?)` -- returns total USD for a day

**Side Effects:** Writes to `cost_log` SQLite table.

## Technical Approach

`src/costs/pricing.ts` exports a `PRICING` map keyed by model ID string, with `{ input: number, output: number }` rates per 1M tokens. `src/costs/index.ts` exports the three functions. Cost calculation: `(inputTokens * pricing.input + outputTokens * pricing.output) / 1_000_000`. Uses existing `insertCostEntry` and `getDailySpend` from `src/db/index.ts`. The `getSessionCost` function queries the DB directly since `src/db/index.ts` doesn't have a session cost aggregation query yet -- add one.

Pricing values should match current published rates at time of implementation. Use comments noting the source URL for each price.
