# Task 6.8 -- Status and Agents Commands

## Objective

Implement the `/status` and `/agents` informational commands that display active sessions, pending gates, daily spend, and a placeholder agent list.

## Context

The design doc defines `/status` as showing current system state (active sessions, pending HITL gates, daily spend) and `/agents` as listing scheduled agents. Since the agent scheduling system is Phase 8, `/agents` is a placeholder in this phase.

These are read-only commands with no side effects beyond sending Telegram messages. They query existing DB functions and gate state.

Depends on 6.4 (dispatch). Blocks 6.9.

## Requirements

- [ ] `StatusDeps` interface with DI:
  - `db: Database`
  - `sendMessage: (chatId: number | string, text: string, options?: { parseMode?: string }) => Promise<void>`
  - `chatId: number`
  - `getPendingGatesFn?: () => HitlGate[]` -- override for testing (defaults to `getPendingGatesList` from gates)
- [ ] `handleStatus(deps): Promise<void>`
  - Active sessions: query `getSessionsByStatus(db, "running")` from `src/db/index.ts`
  - Pending gates: query `getPendingGatesList()` from `src/gates/index.ts`
  - Daily spend: query `getDailySpend(db)` from `src/costs/index.ts` and `getDailySpendByService(db)` from `src/db/index.ts`
  - Format as HTML message with sections
- [ ] `handleAgents(deps): Promise<void>`
  - Placeholder: "No scheduled agents configured. Agent scheduling is Phase 8."
- [ ] `formatStatusMessage(sessions, gates, spend, spendByService): string` -- formats HTML
- [ ] Exported from `src/intake/index.ts` barrel

## Interface Contract

**Inputs:** None (triggered by command dispatch).

**Outputs:** Formatted HTML Telegram message.

**Side Effects:** Sends Telegram message. Read-only DB queries.

## Technical Approach

```typescript
// src/intake/status.ts
import type { Database } from "bun:sqlite";
import { getSessionsByStatus } from "../db/index.ts";
import { getDailySpend } from "../costs/index.ts";
import { getDailySpendByService } from "../db/index.ts";
import { getPendingGatesList } from "../gates/index.ts";
import type { HitlGate } from "../gates/types.ts";
import type { Session } from "../db/types.ts";

export function formatStatusMessage(
  sessions: Session[],
  gates: HitlGate[],
  dailySpend: number,
  spendByService: Array<{ day: string; service: string; total: number }>,
): string {
  const lines: string[] = [];

  lines.push("<b>System Status</b>");
  lines.push("");

  // Active sessions
  lines.push(`<b>Active Sessions:</b> ${sessions.length}`);
  for (const s of sessions.slice(0, 5)) {
    lines.push(`  - ${s.task_id ?? s.id} (${s.agent}, ${s.model})`);
  }
  if (sessions.length > 5) lines.push(`  ... and ${sessions.length - 5} more`);
  lines.push("");

  // Pending gates
  lines.push(`<b>Pending Gates:</b> ${gates.length}`);
  for (const g of gates.slice(0, 5)) {
    lines.push(`  - ${g.id} (${g.gate_type})`);
  }
  lines.push("");

  // Daily spend
  lines.push(`<b>Daily Spend:</b> $${dailySpend.toFixed(2)}`);
  for (const entry of spendByService) {
    lines.push(`  - ${entry.service}: $${entry.total.toFixed(2)}`);
  }

  return lines.join("\n");
}

export async function handleStatus(deps: StatusDeps): Promise<void> {
  const sessions = getSessionsByStatus(deps.db, "running");
  const gates = (deps.getPendingGatesFn ?? getPendingGatesList)();
  const dailySpend = getDailySpend({ db: deps.db });
  const spendByService = getDailySpendByService(deps.db);

  const message = formatStatusMessage(sessions, gates, dailySpend, spendByService);
  await deps.sendMessage(deps.chatId, message, { parseMode: "HTML" });
}

export async function handleAgents(deps: StatusDeps): Promise<void> {
  await deps.sendMessage(
    deps.chatId,
    "No scheduled agents configured. Agent scheduling is Phase 8.",
  );
}
```
